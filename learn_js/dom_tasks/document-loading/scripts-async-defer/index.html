Theory

Scripts with defer never blocks the page

Scripts with defer always executing, when DOM is ready, but before DOMContentLoaded

Scripts with defer intended only for external script only (ignored if script with src)

dynamically added scripts behave like async scripts

defer is used for scripts, that require access to the entire DOM and/or order of execution is important

asyncis good for independent script, for example counters or ads, the relevant orders of execution is irrelevant

  • content of the page displays immediately
  • DOMContentLoaded can appear before or after async script
  • scripts are executed in order the are loaded
  • async scripts are useful for adding to the page external scripts
